# Финиковый Двор — продающий редизайн v2 («Editorial market»)

Дата: 2026-07-06. Утверждён пользователем (вариант A).

## Цель

Страница должна продавать и вызывать доверие с первого экрана, оставаясь в Monad design system (parchment, serif 400, mono UI, один Lake Blue CTA, пилл-формы, 1px ash, без теней).

## Отклонение от канона

Monad — «без фотографий». Для еды фото критичны для продаж (решение пользователя: стоковые с Unsplash/Pexels, самостоятельный хостинг в `/images`, временно до своих). Фото всегда внутри скруглённых карточек (radius ≥ 16px) с 1px ash-рамкой.

## Структура страницы (сверху вниз)

1. **Announcement bar** — ink-фон: «Доставка по Казахстану · Оплата Kaspi».
2. **Header** — как сейчас (serif-лого, mono-меню, чёрная пилл-корзина).
3. **Hero, 2 колонки** — слева serif-заголовок (clamp до 80px), mono-подзаголовок 20px graphite, кнопки: Lake Blue «Смотреть каталог ▸» + ghost «Как заказать». Справа фото фиников в карточке 40px radius, позади градиентный ваш (coral→sky→mint, blur 75px).
4. **Строка доверия** — mono 12-14px uppercase, элементы через ash-разделители: «7 сортов · ручной отбор · поставки еженедельно · оплата Kaspi».
5. **Хит продаж** — широкая periwinkle-карточка (Elevated Feature Card): Меджул, фото справа, serif-заголовок 32-40px, цена, кнопка «В корзину» (off-black fill — Lake Blue занят hero CTA).
6. **Каталог** — карточки с фото (aspect ~4:3, object-fit cover), serif-название 24px, mono-описание 14px, цена/вес, ghost-пилл «В корзину».
7. **«Почему мы»** — 3 feature-карточки: свежесть партий, прямые поставки с Ближнего Востока, ручной отбор. 1px ash, 32px radius, serif-заголовок 24px.
8. **Отзывы** — 3 карточки: mono-текст, serif-имя, город. Правдоподобные заглушки, помечены в коде комментарием «заменить на реальные».
9. **Доставка и оплата** — 2 колонки: сроки (Алматы/Астана/Казахстан) + оплата Kaspi (перевод/счёт), mono.
10. **FAQ-аккордеон** — 4-5 вопросов (хранение, срок годности, опт, доставка), serif-вопрос 24px, ash bottom-border, `<details>` без JS.
11. **Финальный CTA** — ink-фон, serif-заголовок parchment-цветом, белая ghost-кнопка к каталогу.
12. **Футер** — контакты, © 2026.
13. **Плавающая кнопка** (fixed, справа внизу) — off-black пилл «Корзина + счётчик»; резерв под будущего ИИ-консультанта «Мирзаким» (станет чатом).

## Фото

~8 бесплатных фото (7 товаров + hero/хит) с Unsplash/Pexels (лицензии позволяют коммерческое использование без атрибуции). Скачать в `/images`, оптимизировать размер (~1200px по длинной стороне), прописать в `js/products.js` поле `image`. Подарочный набор/шоколад — ближайшие подходящие кадры.

## Технически

- Тот же стек: index.html + css/style.css + js (без фреймворков).
- JS-логика корзины/заказа не меняется; renderProducts уже поддерживает `image`.
- Хит продаж — статичный HTML-блок с data-add кнопкой (id товара 1).
- Адаптив: hero и хит складываются в колонку < 900px; сетки карточек — auto-fill.
- Один Lake Blue CTA на страницу (hero). Cart submit — Lake Blue внутри панели корзины (отдельный контекст, как в v1).

## Критерии готовности

- Все 13 блоков на месте, фото загружаются локально (без хотлинков).
- Корзина: добавление из каталога И из хита продаж работает, заказ отправляется.
- Проверка в preview: скриншоты, computed styles соответствуют системе, консоль без ошибок.
- Push в main → Vercel deploy.
